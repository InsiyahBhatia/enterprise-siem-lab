# Windows Event Monitoring & Threat Hunting Dashboard using Splunk

## Project Overview

This project demonstrates the deployment of Splunk Enterprise and Splunk Universal Forwarder for centralized Windows log monitoring. Windows Security Event Logs and Sysmon logs were collected from a Windows Server and forwarded to Splunk for security monitoring, log analysis, threat hunting, and dashboard creation.

The project focuses on monitoring authentication activity, privilege assignments, process execution events, and overall Windows security events using Search Processing Language (SPL).

# SPL Commands Used

## 1. stats

### Theory

The `stats` command is used to aggregate and summarize data.

### Example

```spl
EventCode=4624 OR EventCode=4673 OR EventCode=4672
| stats count by Account_Name
```

### Result

Counts how many events were generated by each account. 

---

## 2. table

### Theory

Displays only selected fields.

### Example

```spl
EventCode=4624
| table _time Account_Name host
```

### Result

Shows:

* Time
* Username
* Hostname

for every successful login. 

---

## 3. sort

### Theory

Sorts results ascending or descending.

### Example

```spl
EventCode=4624
| stats count by Account_Name
| sort - count
```

### Result

Displays the most active users first. 

---

## 4. where

### Theory

Filters results after aggregation.

### Example

```spl
EventCode=4624
| stats count by Account_Name
| where count>10
```

### Result

Shows only users with more than 10 successful logins. 

---

## 5. eval

### Theory

Creates new calculated fields.

### Example

```spl
EventCode=4624
| stats count by Account_Name
| eval Risk=if(count>100,"HIGH","LOW")
```

### Result

Assigns a risk category to users based on login volume. 

---

## 6. timechart

### Theory

Used for time-based visualizations.

### Example

```spl
EventCode=4624 OR EventCode=4673
| timechart count by EventCode
```

### Result

Shows authentication activity over time. 

---

## 7. search

### Theory

Filters events based on keywords or field values.

### Example

```spl
search Account_Name=Administrator
```

### Result

Displays all Administrator-related events. 

---

## 8. fields

### Theory

Displays only specified fields.

### Example

```spl
EventCode=4688
| fields host New_Process_Name
```

### Result

Shows process execution information.


Shows how users flow into successful logins, failed logins, and privileged logons.
